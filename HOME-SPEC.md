# Home 页面重构 Spec — 方案A：周条 + 下拉月历

## 概述
将当前 DailyView.tsx 重构为真正的 Home 页面，采用"周条 + 下拉展开月历"的布局。

## 布局结构（从上到下）

### 1. Header（保持不变）
- Logo + "拱卒" + 用户头像/菜单
- 菜单内容不变（错题本、切换用户、主题、退出）

### 2. 周条（Week Strip）— 默认收起状态
- 一行7天，显示当前周（周一~周日）
- 每天一个圆形格子，显示日期数字
- 圆形底色表示状态：
  - 🟢 绿色：当天全部完成且准确率 ≥ 80%
  - 🟡 黄色/琥珀色：部分完成 或 准确率 < 80%
  - 🔴 红色边框：有题但没做
  - 无色/默认：没有题目
  - 灰色文字：未来日期
- 今天的圆形有额外蓝色边框高亮
- 点击某天 → 下方内容区切换到那天的题目
- 周条下方有一个小箭头 ▼ 或手势提示，表示可以展开

### 3. 月历面板（下拉展开）
- 点击周条下方的 ▼ 按钮 或 下拉手势 → 展开完整月历
- 月历格子风格与周条一致（同样的颜色标记）
- 左右箭头切换月份
- 点击某天 → 收起月历 + 下方切换到那天的题目
- 再次点击 ▲ → 收起月历回到周条

### 4. 今日/选中日期的练习内容
- 标题：显示选中日期（"今天 4月3日 星期四" 或 "3月15日 星期六"）
- 题目列表卡片（和现在的样式基本一致）
  - 图标 + tag名 + 题数
  - 右侧显示完成状态（✅ 85% / ⏰ 未完成）
- 没有题时显示空状态 "这天没有作业"

### 5. 错题本摘要（始终显示，不管有没有错题）
- 有错题时：
  - 📕 红色图标
  - "错题本 · X个知识点待复习"
  - 右侧 [错题重做] 按钮（仅今天显示，且每天限一次）
  - 点击进入 /mistakes 页面
- 没错题时：
  - 📗 绿色图标
  - "错题本 · 暂无错题，继续加油💪"
  - 点击仍可进入 /mistakes 页面（看已掌握列表）

### 6. 连续打卡（可选，放在周条和内容之间）
- 一行小字："已连续练习 X 天 🔥"
- 计算逻辑：从今天往前数连续有"全部完成"的天数
- 如果今天还没做完，显示 "昨天完成了 X 天连续练习"

## 需要新增的 API

### GET /api/status/monthly?userId=cyan&month=2026-04
返回某月每天的完成摘要：
```json
{
  "userId": "cyan",
  "month": "2026-04",
  "days": {
    "2026-04-01": { "quizCount": 3, "allCompleted": true, "accuracy": 90 },
    "2026-04-02": { "quizCount": 2, "allCompleted": false, "accuracy": 67 },
    "2026-04-03": { "quizCount": 0 }
  },
  "streak": 5
}
```
- 只返回有数据的天
- streak：从今天往前数的连续完成天数

## 技术要求
- 继续用 React + Tailwind + lucide-react
- 暗色模式全覆盖
- 月历数据按月请求一次，切换日期时不重复请求（同月内）
- 周条数据从月历数据中取
- 移动端优先，格子大小适配手机屏幕
- 动画：月历展开/收起用 transition（height + opacity）
- 路由不变：`/:userId/today`、`/:userId/:date`、`/:userId/home` 都指向这个组件

## 现有代码结构
- 前端：`src/pages/DailyView.tsx`（重构这个文件）

- 主题：`src/lib/theme.ts`

## 保留
- Header 的用户菜单（头像、切换用户、主题、退出）
- 题目卡片的样式和交互
- Quiz 页面的跳转逻辑（已修复为返回对应日期）

## 注意事项
- `daily_quizzes` 表没有 source/status 列
- API 需要兼容没有 date 参数的情况（返回所有日期）
- 手机上月历格子要够大（至少 36px），方便点击
- 不要破坏现有的 Quiz.tsx、Mistakes.tsx、Result.tsx 等页面
