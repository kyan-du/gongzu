## 1. 游戏概述

**宫格记忆**是一种「记忆 + 规律发现 + 推理」的综合训练游戏。

每题分两个阶段（4 屏），难度递进：
- 阶段一：3×3 矩阵，1 格留空 → 观察规律 + 记忆 → 选出 4 个图案
- 阶段二：3×3 矩阵，2 格留空 → 观察规律 + 记忆 → 选出 8 个图案

题目由程序自动生成。固定难度，不做升降级。

两者同属 `memory_game` 模块，是并列的两种游戏类型：

```

├── matryoshka（套娃记忆）— 序列记忆，颜色链
└── grid（宫格记忆）— 规律发现 + 关联记忆，图案宫格
```

- 🔲 宫格记忆

路由：`/:userId/memory` → 选择页 → `/:userId/memory/matryoshka` 或 `/:userId/memory/grid`

## 3. 游戏流程

### 每题 = 4 屏，每屏 90 秒倒计时

| 屏 | 阶段 | 内容 |
|----|------|------|
| 1 | 一 | **看题**：3×3 矩阵，1 格为 ?（第 3 列），观察规律 |
| 2 | 一 | **答题**：从备选池选出 ?₁?₂?₃?₄ |
| 3 | 二 | **看题**：3×3 矩阵，2 格为 ?（第 3 列），观察规律 |
| 4 | 二 | **答题**：从备选池选出 ?₁~?₈ |

### 3.1 阶段一（屏 1 + 屏 2）

**屏 1 — 看题**：
```
┌────┬────┬────┐
│ A1 │ A2 │ A3 │
├────┼────┼────┤
│ B1 │ B2 │ B3 │    每格 = 2×2 小宫格（4 个图案）
├────┼────┼────┤
│ C1 │ C2 │ ?  │    第 3 列 1 格留空
└────┴────┴────┘
         ⏱ 90s
```
- 8 格有内容（含可能的空白/裂图），1 格为 ?
- 玩家观察行内第 1 宫 + 第 2 宫之间的变换规律
- 倒计时结束进入答题

**屏 2 — 答题**：
- 备选图案池（正确 + 近似干扰项）
- 按 ?₁→?₂→?₃→?₄ 顺序选出
- 倒计时结束按已选提交

### 3.2 阶段二（屏 3 + 屏 4）

**屏 3 — 看题**：
```
┌────┬────┬────┐
│ A1 │ A2 │ A3 │
├────┼────┼────┤
│ B1 │ B2 │ ?  │    第 3 列 2 格留空
├────┼────┼────┤
│ C1 │ C2 │ ?  │
└────┴────┴────┘
         ⏱ 90s
```
- 7 格有内容，2 格为 ?
- 其中一个 ? 是阶段一推理过的（需要记住）
- 倒计时结束进入答题

**屏 4 — 答题**：
- 从备选池选出 ?₁~?₈（2 格 × 4 图案）
- 倒计时结束提交

### 递进关系
- 阶段一：记 8 格，推理填 1 格（4 图案）
- 阶段二：记 7 格，推理填 2 格（8 图案），其中一格依赖阶段一记忆
- 关联记忆：前面的结果要一直记着

## 4. 矩阵生成逻辑

### 4.1 行规则

每行独立推导，三行共用同一套规则：

```
行 1: [A1] + [A2] → 合并策略 → [A3]
行 2: [B1] + [B2] → 合并策略 → [B3]
行 3: [C1] + [C2] → 合并策略 → [C3]
```

无列规则。? 固定在第 3 列。

### 4.2 生成步骤

1. **选规则**：随机选一组（元素变换 + 位置变换 + 合并策略）
2. **选素材**：根据规则选主题组（有朝向 / 对称均可）
3. **生成第 1 列**：随机填充 3 个 2×2 宫格（可含空白/裂图）
4. **生成第 2 列**：随机填充 3 个 2×2 宫格
5. **推导第 3 列**：按合并策略逐位置计算
6. **生成干扰项**：对正确答案做微调
7. **选隐藏格**：阶段一隐藏 1 格，阶段二隐藏 2 格

## 5. 变换规则体系

### 5.1 元素变换（单个图案自身的变化）

**形态变换**（5 种，互斥选一）：

| # | 变换 | 说明 |
|---|------|------|
| 1 | 旋转 90° | 顺时针 |
| 2 | 旋转 180° | |
| 3 | 旋转 270° | = 逆时针 90° |
| 4 | 水平镜像 | 左右翻转 |
| 5 | 垂直镜像 | 上下翻转 |

**尺寸变换**（独立开关，可与形态叠加）：

| # | 变换 |
|---|------|
| 6 | 缩放（变小） |

叠加示例：旋转 90° + 变小、水平镜像 + 变小

### 5.2 位置变换（2×2 内 4 个位置的排列变化）

位置编号：
```
1 | 2
-----
3 | 4
```

| # | 变换 | 效果 |
|---|------|------|
| 1 | 顺转 90° | 1→2→4→3→1 |
| 2 | 旋转 180° | 1↔4，2↔3 |
| 3 | 逆转 90° | 1→3→4→2→1 |
| 4 | 左右镜像 | 1↔2，3↔4 |
| 5 | 上下镜像 | 1↔3，2↔4 |
| 6 | 主对角交换 | 1↔4 |
| 7 | 副对角交换 | 2↔3 |

### 5.3 特殊元素（不参与变换）

| 元素 | 说明 |
|------|------|
| 空白 | 格子为空 |
| 裂图 | 图片占位符（自定义 SVG，类似经典图片加载失败图标） |

## 6. 合并策略（两个宫格对应位置合并）

每道题定义一套合并策略，覆盖所有可能情况：

### 6.1 两个都是正常图案

| 情况 | 可选结果 |
|------|---------|
| 相同 | 保留 / 空白 / 裂图 |
| 不同 | 第1宫 / 第2宫 / 空白 / 裂图 |

### 6.2 一个特殊 + 一个正常

| 情况 | 可选结果 |
|------|---------|
| 空白 + 非空白 | 非空白 / 空白 |
| 裂图 + 非空白 | 非空白 / 裂图 |

### 6.3 两个都是特殊

| 情况 | 可选结果 |
|------|---------|
| 空白 + 空白 | 空白 / 裂图 |
| 裂图 + 裂图 | 裂图 / 空白 |
| 空白 + 裂图 | 空白 / 裂图 |

## 7. 图案素材

### 7.1 分类

素材按主题组管理，每题从一个组内选取：

| 主题 | 示例 | 对称性 |
|------|------|--------|
| 动物 | 🐶🐱🐰🐻🦊🐼🐸🐷 | 有朝向 |
| 水果 | 🍎🍐🍊🍋🍌🍉🍇🍓 | 对称 |
| 天气 | ☀️🌙⭐☁️❄️⚡🌈☄️ | 混合 |
| 交通 | 🚗🚕🚌✈️🚀🛵🚲⛵ | 有朝向 |
| 食物 | 🍕🍔🍩🎂🍦🧁🍪🍫 | 对称 |
| 运动 | ⚽🏀🎾🏐🎱🏈⚾🥎 | 对称 |
| 昆虫 | 🪰🕷️🦀🐛🦗🐞🐝🦋 | 有朝向 |
| 数字 | 1 2 3 4 5 6 7 8 9 | 混合（6/9 有朝向）|
| 字母 | b d p q A B R P | 有朝向 |
| 汉字 | 大小上下左右东西 | 有朝向 |
| 形状 | ▲ ◆ ► ◄ ▼ | 有朝向 |

### 7.2 选材规则

- 题目含旋转/镜像变换 → 必须从**有朝向**的组选
- 题目只有缩放/保留变换 → 两种组都可用，素材范围更大

## 8. 备选项生成

### 8.1 来源
备选池中的图案**全部来自矩阵中已出现的图案**，不会出现矩阵中没有的图案。

### 8.2 干扰项生成（近似项）
对正确答案做微调：

| 方式 | 示例 |
|------|------|
| 换形态 | 正确旋转 90°，干扰给 180° |
| 换尺寸 | 正确变小，干扰给正常大小 |
| 换图案 | 正确 🐶，干扰给矩阵中出现过的 🐱 |
| 换特殊 | 正确空白，干扰给裂图 |

### 8.3 规则
- 同一个备选项可以选多次
- 备选池约 15-18 个
